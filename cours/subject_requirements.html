📄 Rapport d'Analyse : Sujet Push_swap (v10.1)

Ce rapport détaille exhaustivement toutes les exigences imposées par le sujet officiel de l'école 42.

1. Règles Générales et Restrictions C Critique

  • Langage : C uniquement.
  • Norme : Le code doit passer la Norminette à 100% (incluant les bonus si présents).
  • Stabilité : Aucun crash inattendu (Segfault, bus error, double free). Sanction : Note = 0.
  • Gestion mémoire : AUCUNE fuite mémoire (memory leak) n'est tolérée sur la heap.
  • Fonctions externes autorisées : read, write, malloc, free, exit.
  • Libft : Autorisée (doit inclure son propre Makefile et être compilée par le Makefile principal).
  • Interdictions formelles : Variables globales strictement interdites.

2. Exigences du Makefile

  • Flags obligatoires : -Wall -Wextra -Werror avec cc.
  • Règles obligatoires : $(NAME), all, clean, fclean, re.
  • Relink : Le Makefile ne doit pas relink inutilement.
  • Bonus : Doivent être dans des fichiers _bonus.{c,h} et compilés via une règle bonus séparée.

3. Fonctionnement du Programme push_swap

Le programme reçoit en paramètre la pile a sous forme d'entiers. Le premier argument est le sommet de la pile.

Règles d'affichage et de sortie :

  • Le but est de trier la pile avec le plus petit nombre possible d'opérations.
  • Les instructions doivent être séparées par un \n et rien d'autre.
  • Cas aucun paramètre : Le programme s'arrête discrètement sans RIEN afficher.
  • Cas d'erreur : Afficher Error\n sur la Sortie d'Erreur Standard (stderr / fd 2).
    • Exemples de déclenchement : Entrée non numérique, overflow/underflow d'entier (limites INT), présence de doublons.

Set d'instructions autorisé :

sa, sb, ss, pa, pb, ra, rb, rr, rra, rrb, rrr.

Pour valider le projet, l'algorithme doit trier efficacement :

⚠️ Note : L'éligibilité aux bonus exige la note parfaite (100% / note max) sur les benchmarks.

5. Requirement Spécial : Le README.md

Le sujet v10.1 introduit une nouvelle exigence stricte concernant le dépôt git :

  • La première ligne du README doit être (en italique) : This project has been created as part of the 42 curriculum by [login1].
  • Section "Description" : Présentation rapide du projet.
  • Section "Instructions" : Comment compiler et lancer.
  • Section "Resources" : Références utilisées ET Description de la façon dont l'IA a été utilisée (si d'application).

6. Partie Bonus : Le programme checker

  • Il prend la pile a en arguments (avec les mêmes gestions d'erreurs que push_swap).
  • Il lit sur l'entrée standard (stdin) les instructions une par une.
  • Si à la fin de la lecture, la pile a est triée et b est vide : il affiche OK\n sur stdout.
  • Sinon, il affiche KO\n sur stdout.
  • En cas d'erreur (format argument, ou instruction inexistante lue) : Error\n sur stderr.
