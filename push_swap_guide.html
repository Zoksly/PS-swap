Small Sorters

O(1) - Hardcode

Quand ?

2 a 5 elements.

Comment ?

  • 2 elem : un seul sa si pas trie
  • 3 elem : ft_threefast - toutes les permutations de 3 elements (max 2 ops)
  • 4-5 elem : ft_fivast - push les plus petits vers b, trie les 3 restants, puis pa

Performance

3 elem : max 2 ops | 5 elem : max ~12 ops

Turk Sort (mode --simple)

O(n^2) - Cost-based

Quand ?

Mode --simple. Meilleur algo pour les listes moyennes/grandes. C'est l'algo qui devrait etre utilise par defaut pour atteindre les benchmarks du sujet.

Comment ?

  1. Push 2 elements dans b
  2. Pour chaque element de a, calcule le cout pour le placer dans b a sa position correcte
  3. Optimise avec rr/rrr quand les rotations vont dans le meme sens
  4. Push le moins couteux
  5. Trie les 3 restants dans a avec ft_threefast
  6. Push tout de b vers a a la bonne position
  7. Rotation finale pour mettre le min en haut

Performance attendue

100 elem : ~550-650 ops | 500 elem : ~4500-5500 ops

Selection Sort (Simple)

O(n^2)

Quand ?

Disorder < 20% (mode adaptatif). Liste presque triee.

Comment ?

  1. Trouve l'index du minimum dans a
  2. Le rotate vers le haut (ra ou rra selon la position)
  3. Push vers b
  4. Repete jusqu'a ce que a soit vide
  5. Push tout de b vers a

Performance

Tres bon sur listes presque triees, mais O(n^2) sur listes desordonnees. ~2000+ ops pour 100 elem en general.

Chunk Sort (Medium)

O(n * sqrt(n))

Quand ?

20% ≤ disorder < 50% (mode adaptatif).

Comment ?

  1. Copie la stack dans un tableau trie
  2. Divise en chunks (taille adaptee selon n)
  3. Push chaque chunk de a vers b
  4. Reconstruit a en prenant toujours le max de b

Performance

100 elem : ~800-1200 ops | 500 elem : ~6000-9000 ops selon la taille des chunks.

Radix Sort (Complex)

O(n * log n)

Quand ?

Disorder ≥ 50% (mode adaptatif). Liste tres desordonnee.

Comment ?

  1. Copie la stack dans un tableau trie pour obtenir les index
  2. Pour chaque bit (du LSB au MSB) :
  3. - Si le bit de l'index est 1 : ra
  4. - Si le bit est 0 : pb
  5. Push tout de b vers a apres chaque passe de bit

Performance

100 elem : ~1200-1600 ops | 500 elem : ~5500-8000 ops. Regulier mais pas optimal pour les benchmarks du sujet.
